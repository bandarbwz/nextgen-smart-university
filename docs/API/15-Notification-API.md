# Notification API

## Purpose

This document defines the Notification REST APIs for the NextGen Smart University Platform.

The Notification API manages real-time notifications, announcements, email notifications, push notifications, SMS notifications, user notification preferences, and notification history across all platform modules.

---

# Base URL

```
/api/v1/notifications
```

---

# Authentication

All endpoints require:

```
Authorization: Bearer <JWT_TOKEN>
```

---

# Content Type

```
Content-Type: application/json
```

---

# Notification APIs

---

## Get Notifications

```
GET /api/v1/notifications
```

---

## Get Notification

```
GET /api/v1/notifications/{id}
```

---

## Send Notification

```
POST /api/v1/notifications
```

### Permissions

- Administrator
- System Services

---

## Update Notification

```
PUT /api/v1/notifications/{id}
```

---

## Delete Notification

```
DELETE /api/v1/notifications/{id}
```

---

## Mark as Read

```
PUT /api/v1/notifications/{id}/read
```

---

## Mark All as Read

```
PUT /api/v1/notifications/read-all
```

---

## Archive Notification

```
PUT /api/v1/notifications/{id}/archive
```

---

## Delete All Notifications

```
DELETE /api/v1/notifications
```

# Validation Rules

Notification

- Title is required.
- Message is required.
- Recipient is required.
- Notification type is required.

Announcement

- Title is required.
- Publish date is required.

Broadcast

- At least one recipient group is required.

# Business Rules

- Notifications are generated automatically by integrated modules.
- Users receive notifications based on their preferences.
- Read notifications remain available in history.
- Broadcast notifications are delivered only to selected user groups.
- High-priority notifications override mute settings.
- All notification delivery attempts are logged.
