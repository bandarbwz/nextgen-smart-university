# Development Workflow

## Purpose

This document defines the official development workflow for the project.

Every developer and AI assistant must follow these steps.

---

# Phase 1 - Documentation

Before writing any code:

- Read README.md
- Read AI Instructions
- Read Project Rules
- Read the related feature documentation
- Understand the requirements

No code should be written before documentation is complete.

---

# Phase 2 - Database Design

Design the database before backend development.

Tasks:

- Create tables
- Define relationships
- Create indexes
- Define constraints
- Review database design

---

# Phase 3 - Backend Development

Develop backend features in small tasks.

Each feature must include:

- Models
- Services
- Controllers
- Routes
- Validation
- Authentication
- Authorization

Test every endpoint before continuing.

---

# Phase 4 - Frontend Development

Develop frontend after backend APIs are completed.

Each page must include:

- Responsive design
- Validation
- Error handling
- Loading state
- Success messages

---

# Phase 5 - Integration

Connect frontend with backend.

Verify:

- API requests
- Authentication
- Data validation
- File uploads
- Realtime communication

---

# Phase 6 - Testing

Test every module.

Testing includes:

- Functional testing
- Validation testing
- Security testing
- Performance testing
- UI testing

Fix all critical bugs before moving forward.

---

# Phase 7 - Deployment

Prepare production environment.

Tasks:

- Environment configuration
- Database setup
- Server configuration
- SSL
- Backup
- Monitoring

---

# AI Workflow

Before every task:

1. Read documentation.
2. Analyze requirements.
3. Generate implementation.
4. Review generated code.
5. Fix detected issues.
6. Wait for approval.

Never continue automatically.

---

# Git Workflow

For every completed task:

- git add .
- git commit
- git push

Use clear and simple commit messages.

---

# Final Rule

Complete one task at a time.

Do not start a new task until the current task is approved.
